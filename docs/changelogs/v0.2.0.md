# v0.2.0

## Release Date

2026-03-12

## Overview

All write operations now use the Datasphere UI's import API (`/deepsea/repository/`) instead of the single-object CRUD endpoints. This enables multi-definition CSN imports in a single request, resolves circular dependency issues between views, and automatically deploys after import. The 12 individual create/update/upsert operations have been removed in favor of a single `importCsn` method.

## Breaking Changes

- **`createView`, `updateView`, `upsertView` removed** — Use `client.importCsn(csn)` instead. Pass the full CSN; no need to specify an object name.
- **`createLocalTable`, `updateLocalTable`, `upsertLocalTable` removed** — Same as above.
- **`createAnalyticModel`, `updateAnalyticModel`, `upsertAnalyticModel` removed** — Same as above.
- **`createReplicationFlow`, `updateReplicationFlow`, `upsertReplicationFlow` removed** — Same as above.
- **`UpsertViewResult`, `UpsertLocalTableResult`, `UpsertReplicationFlowResult`, `UpsertAnalyticModelResult` types removed** — `importCsn` returns `ImportCsnResult` with `{ body: string, objectIds: string[] }`.
- **`extractObject`, `validateCsnFile`, `resolveDependencies` removed** — The `core/csn/` module is deleted entirely. The import API handles full CSN files natively.
- **`preDeps` removed from `DatasphereObjectType`** — No longer needed since `importCsn` handles dependency ordering.

## What's New

### `importCsn` — Unified Write Operation

`client.importCsn(csn)` replaces all create/update/upsert operations. It:

1. Resolves the space UUID via `GET /deepsea/repository/{space}/designObjects` (cached)
2. Posts the full CSN to `POST /deepsea/repository/{space}/objects/` with `{ data: { content, saveAction: "import", space_id } }`
3. Parses object GUIDs from the response
4. Deploys via `POST /dwaas-core/deploy/{space}/objects` with the extracted GUIDs

Supports multi-definition CSN files — views with circular references (e.g. `GLACCOUNTINCHARTOFACCOUNTS` ↔ `GLACCOUNTHIERARCHYNODE`) now import correctly.

### `deployObjects` — Standalone Deploy

`deployObjects(requestor, spaceName, spaceId, objectIds)` is available as a core operation for advanced use cases where import and deploy need to be separated.

### `resolveSpaceId` — Space UUID Lookup

`resolveSpaceId(requestor, spaceName)` discovers the internal UUID for a space name. Called automatically by `BdcClient.importCsn()` and cached for the session.

## What's Removed

| Removed | Replacement |
|---------|-------------|
| 12 write operations (create/update/upsert × 4 types) | `client.importCsn(csn)` |
| `core/csn/` module (extract, validate, resolveDeps) | Not needed — import API accepts full CSN |
| `scripts/upload-views.ts` (single-object uploads) | `scripts/upload-views-batch.ts` |
| `scripts/pool.ts` (concurrency utility) | Not needed — single `importCsn` call per batch |
| `preDeps` in object type registry | Import API handles dependency ordering |

## Technical Details

### Scripts Updated

All upload scripts (`upload-tables.ts`, `upload-flows.ts`, `upload-models.ts`) now merge CSN files into a single multi-definition CSN and call `client.importCsn()` once, instead of uploading objects individually with concurrency pools.

### Tests Updated

Integration tests use `importCsn` for object creation instead of individual create/upsert calls. Test lifecycle: import → read → objectExists → delete → objectExists.
