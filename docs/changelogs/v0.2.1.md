# v0.2.1

## Release Date

2026-03-12

## Overview

Adds a navigator module for browsing Datasphere space contents and a data preview capability for querying live data from deployed views and tables via OData. These features enable AI agents and scripts to programmatically explore a space, discover objects, inspect column metadata, and retrieve filtered data — supporting automated debugging and investigation of Datasphere issues.

## What's New

### Navigator — Browse Space Contents

Three new methods for exploring objects within a Datasphere space:

- **`listObjects(options?)`** — Lists all design objects in the space with auto-pagination (fetches in batches of 500). Supports filtering by object kind, glob patterns on names (e.g. `I_*Replication`), and folder exclusion.
- **`listFolders(parentFolderId?)`** — Lists folders in the space. Pass a parent folder ID to drill into subfolders.
- **`searchObjects(options?)`** — Raw search API access with text queries, kind filtering, folder drill-down, and pagination control.

### Data Preview — Query Live Data via OData

Two new methods for reading data from deployed views and local tables:

- **`getViewColumns(viewName)`** — Fetches column metadata (name, OData type, max length, precision, scale, key status) from a view's `$metadata` endpoint. Useful for discovering available columns before querying.
- **`previewData(viewName, options?)`** — Fetches data rows from a deployed view. Supports:
  - **Column selection** via `select` — return only specific fields
  - **Filtering** via `filter` — OData `$filter` expressions (e.g. `"COMPANYCODE eq '1010' and FISCALYEAR eq '2025'"`)
  - **Pagination** via `top` and `skip`
  - Handles both OData v2 and v4 response formats automatically
  - Strips OData metadata (`__metadata`) from returned rows

Views must have an "Active" deployment status to be queryable.

## Technical Details

### New Modules

- `core/operations/navigator/searchObjects.ts` — Search API wrapper with OData query builder
- `core/operations/navigator/listObjects.ts` — Auto-paginated listing with glob and kind filtering
- `core/operations/navigator/listFolders.ts` — Folder listing with parent drill-down
- `core/operations/navigator/previewData.ts` — OData data access via `/dwaas-core/data-access/instant/`
- `core/operations/navigator/getViewColumns.ts` — EDMX `$metadata` parser for column discovery
- `core/utils/glob.ts` — Simple glob-to-regex converter for client-side name filtering

### New Types

- `SearchObject` — Rich object metadata (name, kind, folder, deployment status, timestamps, etc.)
- `SearchOptions` — Query text, kind filter, folder ID, pagination
- `ListObjectsOptions` — Kind filter, glob pattern, folder exclusion toggle
- `SpaceFolder` — Folder with name, ID, display name, parent reference
- `DataPreviewOptions` — Column select, OData filter, top/skip
- `DataPreviewResult` — Rows as `Record<string, unknown>[]` with optional total count
- `ViewColumn` — Column name, OData type, constraints, key flag

### Integration Tests

11 integration tests covering all navigator and data preview operations against the live Datasphere API.
